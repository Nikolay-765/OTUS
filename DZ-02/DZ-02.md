# ДЗ № 2
## Тема: Работа с mdadm

Задание
- Добавьте в виртуальную машину несколько дисков
- Соберите RAID-0/1/5/10 на выбор
- Сломайте и почините RAID
- Создайте GPT таблицу, пять разделов и смонтируйте их в системе.

## Выполнение задания
Изначально имеем
```
root@server:/# lsblk
NAME                      MAJ:MIN RM  SIZE RO TYPE MOUNTPOINTS
sda                         8:0    0   50G  0 disk
├─sda1                      8:1    0    1M  0 part
├─sda2                      8:2    0    2G  0 part /boot
└─sda3                      8:3    0   48G  0 part
  └─ubuntu--vg-ubuntu--lv 252:0    0   24G  0 lvm  /
sdb                         8:16   0    2G  0 disk
sdc                         8:32   0    2G  0 disk
sdd                         8:48   0    2G  0 disk
sde                         8:64   0    2G  0 disk
sdf                         8:80   0    2G  0 disk
sr0                        11:0    1 1024M  0 rom
```
Зачищаем на дисках суперблоки (исли они есть) и собираем RAID-5 из 4-х дисков + 1 hot spare диск
```
root@server:/# mdadm --zero-superblock --force /dev/sd{b..f}
root@server:/# mdadm --create --verbose /dev/md0 -l 5 -n 4 /dev/sd{b..e}
root@server:/# mdadm /dev/md0 --add /dev/sdf
```
В итоге получили
```
root@server:/# mdadm -D /dev/md0

              Name : server:0  (local to host server)
              UUID : ba0e40e6:e7559bcc:c2f117ad:8b233dd0
            Events : 20

    Number   Major   Minor   RaidDevice State
       0       8       16        0      active sync   /dev/sdb
       1       8       32        1      active sync   /dev/sdc
       2       8       48        2      active sync   /dev/sdd
       4       8       64        3      active sync   /dev/sde

       5       8       80        -      spare   /dev/sdf
```

Убиваем один из дисков RAID'а/
DAID самостоятельно восстановился за счет hot spare диска
```
root@server:/# mdadm /dev/md0 --fail /dev/sdc
root@server:/# mdadm -D /dev/md0

    Number   Major   Minor   RaidDevice State
       0       8       16        0      active sync   /dev/sdb
       5       8       80        1      active sync   /dev/sdf
       2       8       48        2      active sync   /dev/sdd
       4       8       64        3      active sync   /dev/sde

       1       8       32        -      faulty   /dev/sdc
```

Убиваем еще один диск и в результате получаем деградирующий массив.
```
root@server:/# mdadm /dev/md0 --fail /dev/sde
root@server:/# cat /proc/mdstat
Personalities : [raid6] [raid5] [raid4] [raid0] [raid1] [raid10]
md0 : active raid5 sdf[5] sdc[1](F) sdb[0] sdd[2] sde[4](F)
      6282240 blocks super 1.2 level 5, 512k chunk, algorithm 2 [4/3] [UUU_]

root@server:/# mdadm -D /dev/md127

    Number   Major   Minor   RaidDevice State
       0       8       16        0      active sync   /dev/sdb
       5       8       80        1      active sync   /dev/sdf
       2       8       48        2      active sync   /dev/sdd
       -       0        0        3      removed

       1       8       32        -      faulty   /dev/sdc
       4       8       64        -      faulty   /dev/sde
```

Удаляем сломанные диски из массива и зануляем у них суперблоки
```
root@server:/# mdadm /dev/md0 --remove /dev/sd{c,e}
root@server:/# mdadm --zero-superblock --force /dev/sd{c,e}
```
Подключаем к рейду один "новый" диск и он восстанавливается
```
root@server:/# mdadm /dev/md0 --add /dev/sdc
root@server:/# cat /proc/mdstat
Personalities : [raid6] [raid5] [raid4] [raid0] [raid1] [raid10]
md0 : active raid5 sdc[4] sdf[5] sdb[0] sdd[2]
      6282240 blocks super 1.2 level 5, 512k chunk, algorithm 2 [4/4] [UUUU]
```

Перезагружаемся, /dev/md0 сменился на /dev/md127
Создаем на RAID`е таблицу GPT и бъем массив на 5 разделов.
Разделы размечаем под ext4, создаем директории под точки монтирования и выясняем UUID новых разделов. 

```
root@server:/home/user# parted -s /dev/md127 mklabel gpt
root@server:/home/user# parted /dev/md127 mkpart primary ext4 0% 20%
root@server:/home/user# parted /dev/md127 mkpart primary ext4 20% 40%
root@server:/home/user# parted /dev/md127 mkpart primary ext4 40% 60%
root@server:/home/user# parted /dev/md127 mkpart primary ext4 60% 80%
root@server:/home/user# parted /dev/md127 mkpart primary ext4 80% 100%
root@server:/home/user# mkfs.ext4 /dev/md127p1
root@server:/home/user# mkfs.ext4 /dev/md127p2
root@server:/home/user# mkfs.ext4 /dev/md127p3
root@server:/home/user# mkfs.ext4 /dev/md127p4
root@server:/home/user# mkfs.ext4 /dev/md127p5
root@server:/home/user# mkdir -p /media/raid/part{1..5}
root@server:/home/user# lsblk -f
```

Добавляем в /etc/fstab следующие строки:
```
UUID=f50eccbc-9075-49c7-b5fb-7d4561103fc8 /mnt/raid/part1 ext4 defaults 0 2
UUID=69ff0f37-a37d-4307-85da-e94e276d0032 /mnt/raid/part2 ext4 defaults 0 2
UUID=1ddf90dd-f833-42b7-a835-1ed1f7b45a06 /mnt/raid/part3 ext4 defaults 0 2
UUID=35e55662-93f6-434a-b0b5-d50ac8d75ce8 /mnt/raid/part4 ext4 defaults 0 2
UUID=89de9142-f63a-4376-a862-5c5ace4fa93a /mnt/raid/part5 ext4 defaults 0 2
```
