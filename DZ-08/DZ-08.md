# ДЗ № 8
## Тема: Systemd
Задание:
1. Написать service, который будет раз в 30 секунд мониторить лог на предмет наличия ключевого слова (файл лога и ключевое слово должны задаваться в /etc/default).
2. Установить spawn-fcgi и создать unit-файл (spawn-fcgi.sevice) с помощью переделки init-скрипта (https://gist.github.com/cea2k/1318020).
3. Доработать unit-файл Nginx (nginx.service) для запуска нескольких инстансов сервера с разными конфигурационными файлами одновременно.

## Пунк №1

Создаем файл с конфигурацией /etc/default/sshscanlog
```
Pattern="\"Accepted password|Failed password\""
LogFile="/var/log/sshscanlog.log"

```
Пишем скрипт, который из логов достает информиацию о попытках подключения (успешных и неудачных) к системе по SSH за последние 31 сек. (1 сек., лишняя что бы точно ничего не пропустить ) \
/opt/sshscanlog.sh 
```
#!/bin/bash

Pattern=$1
LogFile=$2

journalctl -u ssh --since "31 second ago" | grep -E "$Pattern" >> $LogFile

exit 0
```
Даем скрипту права на исполнение
```
root@server:/opt# chmod +x ./sshscanlog.sh
```
Создаем юнит для сервиса (запуск скрипта) \
/lib/systemd/system/sshscanlog.service
```
[Unit]
Description=Looking for SSH connections

[Service]
Type=oneshot
EnvironmentFile=/etc/default/sshscanlog
ExecStart=/opt/sshscanlog.sh $Pattern $LogFile
```
Создаем юнит для таймера (дергает сервис каждые 30 сек.) \
/lib/systemd/system/sshscanlog.timer
```
[Unit]
Description=Run sshscanlog service every 30 second

[Timer]
OnBootSec=10
OnUnitActiveSec=30
Unit=sshscanlog.service

[Install]
WantedBy=multi-user.target
```
На всякий случай делаем
```
root@server:/lib/systemd/system# systemctl daemon-reload
```
Перезагружаемся и несколько раз подключаемся к VM по SSH \
Смотрим лог

## Пунк №2
Не уверен, что понял смысл этого задания. Полагаю, что нужно запустить spawn-fcgi как сервис с настройками из init-скрипта\
Устанавливаем spawn-fcgi, apache2 и т.д.
```
root@server:/# apt update
root@server:/# apt upgrade
root@server:/# apt install spawn-fcgi php php-cgi php-cli apache2 libapache2-mod-fcgid -y
```
Создаем unit-файл сервиса spawn-fcgi.service
```
oot@server:/# nano /etc/systemd/system/spawn-fcgi.service
```
Записываем в него следующее
```
[Unit]
Description=Spawn-fcgi startup service by Otus
After=network.target

[Service]
Type=simple
PIDFile=/run/spawn-fcgi.pid
EnvironmentFile=/etc/spawn-fcgi/fcgi.conf
ExecStart=/usr/bin/spawn-fcgi -n $OPTIONS
KillMode=process

[Install]
WantedBy=multi-user.target
```
На всякий случай делаем
```
root@server:/# systemctl daemon-reload
```
Создаем конфигурационный файл
```
root@server:/# nano /etc/spawn-fcgi/fcgi.conf
```
И прописываем в него настройки из init-скрипта
```
OPTIONS="-a 127.0.0.1 -p 9000 -u www-data -g www-data -P /run/php_cgi.pid -C 5 -- /usr/bin/php-cgi"
```
Запускаем сервис spawn-fcgi и смотрим, что получилось

## Пунк №3
Устанавливаем nginx и тушим его (в стандартной конфигурации)
```
root@server:/# apt update
root@server:/# apt upgrade
root@server:/# apt install nginx
root@server:/# systemctl stop nginx
root@server:/# systemctl disable nginx
```
На базе стандартной создаем две новых конфигурации для инстансов nginx 
```
root@server:/etc/nginx# cp nginx.conf nginx-first.conf
root@server:/etc/nginx# cp nginx.conf nginx-second.conf
```
В новых конфирурациях добавляем/меняем следующее 
```
pid /run/nginx-first.pid;
server {
       listen 1080;
       root /var/www/first;
       index index.html;
       }
access_log /var/log/nginx/access-first.log;
#       include /etc/nginx/conf.d/*.conf;
#       include /etc/nginx/sites-enabled/*;
```
```
pid /run/nginx-second.pid;
server {
       listen 8080;
       root /var/www/second;
       index index.html;
       }
access_log /var/log/nginx/access-second.log;
#       include /etc/nginx/conf.d/*.conf;
#       include /etc/nginx/sites-enabled/*;
```
Создаем корневые дирректории для инстансов nginx
```
root@server:/var/www# mkdir first
root@server:/var/www# mkdir second
root@server:/var/www# cp -r html/ first
root@server:/var/www# cp -r html/ second
root@server:/var/www# chown root:www-data -R ./*
```
Создаем unit-файл /etc/systemd/system.nginx@.service для запуска копий nginx \
Наполнение unit-файла из методички
```
root@server:/etc/systemd/system# nano nginx@.service
root@server:/etc/systemd/system# systemctl daemon-reload
```
Запускаем копии две nginx с конфигурациями nginx-first.conf и nginx-second.conf
```
root@server:/etc/systemd/system# systemctl start nginx@first
root@server:/etc/systemd/system# systemctl start nginx@second
```
