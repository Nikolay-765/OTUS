# ДЗ № 5
## Тема: Работа с NFS
Задание
1. Запустить 2 виртуальных машины (сервер NFS и клиента).
2. На сервере NFS должна быть подготовлена и экспортирована директория.
3. В экспортированной директории должна быть поддиректория с именем upload с правами на запись в неё.
4. Экспортированная директория должна автоматически монтироваться на клиенте при старте виртуальной машины (systemd, autofs или fstab — любым способом);
5. Монтирование и работа NFS на клиенте должна быть организована с использованием NFSv3.

## Выполнение задания
Server IP - 192.168.0.50/24\
Client IP - 192.168.0.25/24

## Настраиваем NFS сервер
Устанавливаем NFS сервер и проверяем, что он запустился
```
root@server:/# apt install nfs-kernel-server
root@server:/# ss -tulpn | grep -e 111 -e 2049

udp   UNCONN 0      0               [::]:111           [::]:*    users:(("rpcbind",pid=1311,fd=7),("systemd",pid=1,fd=213))
tcp   LISTEN 0      4096         0.0.0.0:111        0.0.0.0:*    users:(("rpcbind",pid=1311,fd=4),("systemd",pid=1,fd=210))
tcp   LISTEN 0      64           0.0.0.0:2049       0.0.0.0:*
tcp   LISTEN 0      4096            [::]:111           [::]:*    users:(("rpcbind",pid=1311,fd=6),("systemd",pid=1,fd=212))
tcp   LISTEN 0      64              [::]:2049          [::]:*
```
Создаем директорию для шары
```
root@server:/# mkdir -p /srv/share/upload
root@server:/# chown -R nobody:nogroup /srv/share
root@server:/# chmod 777 /srv/share/upload/
```
Дописываем в конфигурационный файл /etc/exports строку для экспорта директории шары
```
/srv/share/ 192.168.0.0/24(rw,sync,all_squash,no_wdelay)
```
Oбновляем список экспортируемых директорий и смотрим, что вышло

## Настраиваем NFS клиент
Устанавливаем NFS клиент и создаем директорию для монтирования
```
root@client:/# apt install nfs-common
root@client:/# mkdir -p /mnt/share
```
Добавляем следующую строку в файл /etc/fstab
```
192.168.0.50:/srv/share/ /mnt/share nfs vers=3,proto=tcp,noauto,x-systemd.automount 0 0
```
Применяем новую конфигурацию
```
root@client:/# systemctl daemon-reload
root@client:/# systemctl restart remote-fs.target
```
Проверяем, что вышло\
Перед этим заходим в директорию шары т.к. в соотвествии с noauto,x-systemd.automount она автоматом монтируется только после первого обращения к ее точке монтирования.
```
root@client:/# cd /mnt/share/
root@client:/# mount | grep /mnt/share
systemd-1 on /mnt/share type autofs (rw,relatime,fd=65,pgrp=1,timeout=0,minproto=5,maxproto=5,direct,pipe_ino=11554)
192.168.0.50:/srv/share/ on /mnt/share type nfs (rw,relatime,vers=3,rsize=524288,wsize=524288,namlen=255,hard,proto=tcp,timeo=600,retrans=2,sec=sys,mountaddr=192.168.0.50,mountvers=3,mountport=34037,mountproto=tcp,local_lock=none,addr=192.168.0.50)
```
На первый взгляд все работает. Используется NFSv3 (vers=3)
